2017 Launch

This directory contains data of 2017 launch in three sub-folders, Database, Htdocs and FreeSWITCH.
Applications Deployed in this launch: Baang, Sawaal, Rahemma and Polly.
Telephony Server-Used: FreeSwitch.

Database


Databases of Polly2017, Sawaal2017, Baang2017, and Rahemma2017 in sql files are contained in this (Database) directory.


Description of Sql Files:

baang-2017-06-30.sql Database of Baang, contain tables such as userplayrectable , uservotesofrec , recording table , feedback table , ..
pquiz-2017-06-25.sql Data of Sawaal, tables such as facts , questions , user answers , user likes ..
pollywa-2017-06-25.sql Data of Polly, tables such as call table , request table , user table , ..

The '2017-06-25' sub-string in names of sql files represents the date 2017-06-25 on which the application launch is cluminated.

FreeSWITCH


Telephony server Root Directory.
It contains three sub-folder namely conf, scripts and logs.


conf


Configuration Files of FreeSWITCH. Important config file/folders are: vars.xml , autoload_configs , dialplan and sip_profiles.


vars.xml

System wide variables. Default configurations such as default port and password of FreeSWITCH.
Default Password must be Changed
The default password of FreeSWITCH is defined in: 

<X-PRE-PROCESS cmd="set" data="default_password=Conakry2014"/>


autoload_configs


Configuration files which loaded automatically into FreeSWITCH and contain configuration information for core modules.

Main config files used are: modules.conf.xml and sofia.conf.xml.


modules.conf.xml tells FreeSWITCH which modules to load.


sofia.conf.xml contains a "X-PRE-PROCESS" directive that includes other XML files (within, by default, conf/sip_profiles/) that define one or more "SIP Profiles".


dialplan


XML-based call-routing mechanism. It route dialed/received call to its appropriate endpoint, which is an extension specific to deployed IVR Application. Extensions invokes appropiate Lua script , for example on receiving call invokes IncomingHook.lua, in scripts folder FreeSwitch/scripts
public.xml: This contains configurations for handling calls coming in to FreeSWITCH.


Dialplans configuration is mainly contained in two files: default.xml and public.xml , representing default and public context for handling call requests.


default.xml primary FreeSWITCH Dialplan configuration.


public.xml contains configurations, extensions defined for deployed IVR Applications, for handling calls coming in to FreeSWITCH.


As per default the contexts are called "Public" and "Default".
Everything in the public context is available to everyone, while everything in "default" context are only available to users that have registered with FreeSWITCH. You can define in the user profiles (in the directory) which context they belong to, and you can define the public context in the sofia configuration.


sip_profiles


SIP Profiles allow you to define paths to devices or carriers that may live inside or outside your network. A SIP Profile is a SIP UA (User Agent or an endpoint), which communicates with other SIP endpoints.


Files of Sip Profiles used are: internal.xml and external.xml , representing internal and external sip-profiles.


internal.xml represent the Internal sip_profile, and is used to communicate with devices on local network.


external.xml represent the External sip_profile, and is used to communicate with wateen gateway, service provider.


scripts


Defined as default scripts directory. It contains Lua Scripts which enables IVR Applications ( Baang, Sawaal and Polly ) Master code script to handle incoming\outgoing calls.
Master code script hosted on local XAMP server.


IVR Applications connects to the FreeSWITCH server on the port 8021 (TCP Socket Connection, pass= 'Conakry2014') and sends api commands invoking relevant lua scripts, which render and respond back appropiately to application.


Freeswitch extensions invoke lua scripts that in turn invoke the master code script, for event notification and call control intiation according to different IVR application. For example. FreeSWITCH notify Baang IVR application using IncomingHook.lua, when Baang User make an inbound call.


Using Socket Connections, IVR Applications can :


  • check status isActive.lua
  • make outbound calls, for Polly call.lua , for Baang call_baang.lua , for PQuiz call_pquiz.lua , for Rahemaa call_rem.lua
  • answer incoming calls answer.lua
  • hang up a call hangup.lua
  • reject a call reject.lua
  • record user voice record.lua
  • play sound files ask.lua
  • play with gather input askGather.lua.


logs


cdr_logs.csv having Call Data Records (CDR) logs of calls like calls times and durations.
