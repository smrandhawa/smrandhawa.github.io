2016 Launch

This directory contains data of 2016 launch in three sub-folders, Database, Htdocs and FreeSWITCH.
Applications Deployed in this launch: Baang, Sawaal, and Polly.
Telephony Server-Used: FreeSwitch.

Database


Databases of Polly2016, Sawaal2016, and Baang2016 in sql files are contained in this (Database) directory.


Description of Sql Files:


File                                          Description
baang-2016-02-25.sql Database of Baang, contain tables such as userplayrectable , uservotesofrec , recording table , feedback table , ..
pquiz-2016-02-25.sql Data of Sawaal, tables such as facts , questions , user answers , user likes ..
pollywa-2016-02-25.sql Data of Polly, tables such as call table , request table , user table , ..
pollywasys-2016-02-25.sql Phone-number to encoded-key mapping data, phkey table , of users in deployed applications, Sawaal, Baang and Polly in 2016 launch.

The '2016-02-25' sub-string in names of sql files represents the date 2016-02-25 on which the application launch is cluminated.


Htdocs


Web root of HTTP server.


Description of important Folders, it contains:


Folder                                          Description
Baang Baang code scripts and recording data.
BaangMonitor Baang monitor scripts.
FS Master code scripts, APollyLHR.php, controls user's interaction in calls for deployed applications. It controls calls through Native communication with FreeSWITCH, over the event socket interface.
PollyMonitorWA Polly monitor scripts.
PQuiz Sawaal code scripts and recording data.
PQuizMonitor Sawaal monitor scripts.
wa Audio-editing Praat scripts , Call-logs of all deployed applications Qlogs.rar , and Polly code scripts and recording data.


FreeSWITCH


Telephony server Root Directory.
It contains three sub-folder namely conf, scripts and logs.


conf


Configuration Files of FreeSWITCH.
Important config file/folders are: vars.xml , autoload_configs , dialplan and sip_profiles.
Configuration files are same as is defined in 2017 Launch/FreeSwitch/conf/.
Only dialplan\public.xml is different with added extensions defined, for Rahemma Application.


scripts


Scripts are same as is defined in 2017 Launch/FreeSwitch/scripts/.
The difference is that in 2016 only [call.lua(https://drive.google.com/open?id=0B2rhATWIcB9BZ3BJMTQyaGV5dmc) script is used to make outbound calls for all applications while in 2017 sperate calling script for each applicaiton was defined.

Hence, it does not contain call_baang, call_pquiz and call_rem.


logs


cdr_logs.csv having Call Data Records (CDR) logs of calls like calls times and durations.
